µC-Beschreibungsdateien (ECU-XML Datei)

Damit Labskaus alle einzelnen Variaben aus dem Speicher des Mikrokontrollers auslesen kann, benötigt es die Information wo welche Variable wo zu finden ist. Diese Information ist nur eine von vielen die in der ECU-Beschreibungsdatei abgelegt ist. Eine Auflistung aller Informationen befindet weiter unten.

Eine ECU-XML-Datei kann geladen werden indem in der Menuleiste die Option 'Open ECU-XML-file' ausgwählt wird und anschließend die entsprechende Datei ausgewählt wird. Alle Variablen die in dieser ECU-XML-Datei gefunden werden werden automatisch in dem linken Auswahlfenster angezeigt. Weiterhin werden automatisch die Einstellungen des CCP Masters (dies ist der PC auf dem Labskaus ausgeführt wird) mit den gespeicherten Information aus der ECU-XML-Datei aktualisiert. Dieser bedeutet aktuel eine automatische Anpassung der Stationsadresse und der Byte-Reihenfolge (Endianness) des .

Hinweis: In der Automobilindustrie wird als ECU-Beschreibungsdatei häufig ein sogenanntes a2l-Datei verwendet.Der Aufbau einer solchen a2l-Datei ist in einem Standard fest definiert. Dieser wird jedoch im Augenblick nicht von Labskaus unterstützt. Stattdessen wird ein ECU-Beschreibungsdatei verwendet welche auf einer XML-Datei basiert. Der Grund hierfür ist dass sich die programmatische Umsetzung hiervon sich um einiges leichter gestaltet.

Informationen je ECU (einmalig)

  • Speicherendung (Endianess)
  • Stationsaddresse
  • Version von Stockente

Informationen je Variable (mehrfach)

  • Name der Variable
  • Speicheraddresse
  • Datentyp
  • Einheit (optional)
  • Beschreibung (optional)

Erstellen ein µC-Beschreibungsdatei (ECU-XML Datei)

Zum Erstellen einer µC-Beschreibungsdatei kann das Konsolenprogramm 'Stockente' verwendet werden. Das Programm steht ebenfalls auf der Internetseite http://graugans.ddns.net zur Verfügung wurde genau so wie Labskaus für Windows wie auch für Linux. Zusätzlich sind von diesem Programm auch die Quellen offengelegt worden und über Git-Hub erreichbar, ein Schritt der Labskaus noch nicht gemacht wurde das Codestil hier echt nicht zu empfehlen ist.

Ein paar Stickpunkte zum Programm Stockente und dessen Suchalgorithmus:

  • Aktuell ist das Konsolenprogramm Stockente die einzige Möglichkeit eine µC-Beschreibungsdatei zu erstellen.
  • Das Konsolenprogramm Stockente kann direkt in die Toolchain der µC-IDE (Entwicklungsumgebung für den Mikrokontroller) mit eingebunden werden. Dieses ermöglicht dass z.B. direkt nach jedem kompilieren des Quelltextes auch automatisch die dazugehörige µC-Beschreibungsdatei generiert wird.
  • In Zukunft soll der Benutzer von Labskaus jedoch auch die Möglichkeit haben die ECU-XML-Datei direkt aus dem Programm Labskaus heraus zu erstellen. Deshalb werden die Alogrithmen von Stockente mit dem Release v1.3.1 in Labskaus mit integriert.
